# Berry DB 用户指南

## 快速开始

### 1. 安装
1. 在 VSCode 中打开扩展市场
2. 搜索 "Berry DB"
3. 点击安装

### 2. 创建第一个连接
1. 点击左侧活动栏的 **数据库图标** 🗄️
2. 点击 **+** 按钮
3. 选择数据库类型（例如 SQLite）
4. 填写连接信息：
   - **连接名称**: `MyDatabase`
   - **数据库类型**: SQLite
   - **文件路径**: `C:\mydb.db`
5. 点击 **测试连接** 验证
6. 点击 **保存连接**

### 3. 连接数据库
1. 点击左侧的连接节点
2. 等待连接成功（显示 "连接成功" 提示）
3. 连接节点变为可展开状态

### 4. 浏览数据库
1. 展开连接节点
2. 展开数据库节点
3. 展开 **Tables** 查看所有表
4. 点击表节点查看详细信息：
   - **Columns** - 字段列表
   - **Indexes** - 索引列表
   - **Foreign Keys** - 外键约束
   - **View Data** - 查看数据

### 5. 执行查询
1. 点击左侧的 **📄 New Query** 按钮
2. 或右键点击数据库节点 → **Open Query Editor**
3. 输入 SQL 语句，例如：
```sql
SELECT * FROM users LIMIT 100;
```
4. 点击编辑器标题栏的 **运行** 按钮
5. 查询结果会显示在**右侧面板**

## 功能详解

### 连接管理

#### 新建连接
- 点击 **+** 按钮
- 支持的数据库类型：
  - SQLite - 本地文件数据库
  - PostgreSQL - 关系型数据库
  - MySQL - 关系型数据库
  - Redis - 键值存储

#### 连接/断开
- **连接**: 点击连接节点
- **断开**: 点击连接节点旁的插头图标

#### 编辑连接
1. 右键点击连接节点
2. 选择 **Edit Connection**
3. 修改信息后保存

#### 删除连接
1. 右键点击连接节点
2. 选择 **Delete Connection**
3. 确认删除

### 查询功能

#### 新建查询
- 点击 **📄 New Query** 按钮
- 或使用命令面板: `Ctrl+Shift+P` → `Berry DB: New Query`

#### 执行查询
- **全文执行**: 不选中任何文本，直接点击运行
- **选中执行**: 选中部分 SQL 后点击运行
- **快捷键**: 暂无（计划中）

#### 查询结果
结果会显示在右侧面板，包含：
- **工具栏**: 刷新、复制、导出
- **查询预览**: 显示执行的 SQL
- **数据网格**: DataGrip 风格的表格
- **状态栏**: 行数、列数、执行时间

#### 复制数据
- **复制表头**: 点击工具栏的 📋 表头
- **复制行**: 点击工具栏的 📄 行
- **复制全部**: 点击工具栏的 📑 全部

#### 导出数据
- **CSV**: 点击 📥 CSV
- **JSON**: 点击 📋 JSON
- **INSERT**: 点击 💾 INSERT

### 表操作

#### 查看表数据
1. 右键点击表节点
2. 选择 **View Data**
3. 数据显示在右侧面板

#### 查看表结构
展开表节点查看：
- **Columns** - 所有字段及其类型
- **Indexes** - 所有索引
- **Foreign Keys** - 所有外键约束

### 调试工具

#### 查看连接信息
1. 按 `Ctrl+Shift+P`
2. 输入 `Berry DB: Debug Connections`
3. 查看输出面板中的详细信息

## 常见问题

### 连接失败
**问题**: 点击连接节点后显示 "连接失败"

**解决方案**:
1. 检查数据库服务是否运行
2. 检查连接信息是否正确
3. 点击 **测试连接** 验证配置
4. 查看输出面板中的错误信息

### 右键菜单不显示
**问题**: 右键点击连接节点没有菜单

**解决方案**:
1. 重新加载窗口: `Ctrl+Shift+P` → `Developer: Reload Window`
2. 运行 `Berry DB: Debug Connections` 检查连接状态
3. 确保连接节点的 contextValue 正确

### 查询结果不显示
**问题**: 执行查询后右侧面板没有显示结果

**解决方案**:
1. 检查是否已连接数据库
2. 检查 SQL 语法是否正确
3. 查看右侧面板是否被最小化
4. 重新执行查询

### 删除连接没有反应
**问题**: 右键选择删除连接后没有反应

**解决方案**:
1. 确保右键菜单正常显示
2. 查看输出面板是否有错误
3. 尝试重新加载窗口

## 快捷操作

### 命令面板命令
按 `Ctrl+Shift+P` 后输入:

- `Berry DB: New Query` - 新建查询
- `Berry DB: Add Connection` - 添加连接
- `Berry DB: Refresh Connection` - 刷新连接
- `Berry DB: Debug Connections` - 调试连接
- `Berry DB: Execute Query` - 执行查询

### 右键菜单
- **连接节点**:
  - Edit Connection - 编辑连接
  - Delete Connection - 删除连接
  - Connect/Disconnect - 连接/断开

- **表节点**:
  - View Data - 查看数据
  - View Structure - 查看结构
  - Drop Table - 删除表

- **数据库节点**:
  - Open Query Editor - 打开查询编辑器

## 技巧和提示

### 1. 使用查询历史
- 执行过的查询会自动保存
- 可以在查询历史中重新执行

### 2. 多连接管理
- 可以同时连接多个数据库
- 执行查询时会提示选择连接

### 3. 密码安全
- 密码使用 VSCode 的 SecretStorage 加密存储
- 不会明文保存在配置文件中

### 4. 性能优化
- 大表数据会限制显示前 100 行
- 可以使用 LIMIT 子句控制返回行数

### 5. 错误排查
- 查看输出面板的 Berry DB 日志
- 使用 Debug Connections 命令诊断问题
- 检查 VSCode 开发者工具控制台

## 支持的数据库

### SQLite
- 无需安装额外服务
- 使用本地文件
- 适合小型项目和测试

### PostgreSQL
- 需要安装 PostgreSQL 服务
- 支持高级功能
- 适合生产环境

### MySQL
- 需要安装 MySQL 服务
- 广泛使用
- 适合各种规模项目

### Redis
- 需要安装 Redis 服务
- 键值存储
- 适合缓存和会话管理
